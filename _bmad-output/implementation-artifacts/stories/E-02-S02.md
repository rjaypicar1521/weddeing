# Story E-02-S02 - Email Verification

## Status
Done

## Summary
Implemented email verification workflow end-to-end.

Backend:
- POST /api/v1/auth/email/verify/{id}/{hash} (signed URL validation)
- POST /api/v1/auth/email/resend (Sanctum protected, throttle 6/1m)
- EnsureEmailIsVerified middleware for /api/v1 with explicit auth route exceptions
- Custom verification notification sends frontend verification link
- Resend mailer wired (MAIL_MAILER=resend + RESEND_API_KEY)

Frontend:
- /auth/verify-email page with check-inbox state, resend button, and verification handling
- Redirect to /dashboard?verified=1 on successful verification
- /dashboard success toast-style alert when verified query param exists
- shadcn-style Card/Button/Alert components
- TanStack Query mutation for resend
